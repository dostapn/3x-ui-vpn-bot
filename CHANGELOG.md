## [1.5.0] - 2026-02-02

### 📊 Система отчетности по трафику - замена "ежедневных говноотчетов"

#### Добавлено
- **Новая система отчетности:** Реализован автоматический сбор и анализ трафика клиентов X-UI с ежедневной/недельной/месячной периодичностью.
- **Ежедневные отчеты:** В 00:01 бот сохраняет снимок статистики за прошедший день, отправляет отчет админу и сбрасывает счетчики.
- **Недельный сводный отчет:** По понедельникам формируется агрегированная статистика за 7 дней на основе сохраненных снимков.
- **Месячный отчет:** Каждого 1-го числа месяца - полный отчет за прошедший месяц на основе исторических данных.
- **База данных истории трафика:** Добавлена таблица `traffic_history` для хранения ежедневных снимков с возможностью агрегации по периодам.
- **Планировщик задач:** Интегрирован модуль `APScheduler` для выполнения регулярных задач отчетности.
- **Тестовые скрипты:** Созданы безопасные тесты (`verify_traffic_flow.py`) с изоляцией данных (используется копия БД).
- **Документация для разработчиков:** Создан файл `AGENTS.md` с правилами разработки, тестирования и работы с данными.

#### Изменено
- **bot/utils/scheduler.py:** Создан новый модуль планировщика с функциями отчетности и типизированными данными.
- **bot/database.py:** Добавлены методы работы с историей трафика (`save_traffic_snapshot`, `get_traffic_stats`, `reset_xui_traffic`).

#### Исправлено
- **Стилистические ошибки:** Удалены лишние пробелы в конце строк в SQL-запросах базы данных.

## [1.2.1] - 2026-01-22

### 🐛 Исправлено

- **py3xui SSL поддержка:** Реализована поддержка самоподписанных сертификатов через monkey-patching requests
- **Валидация URL:** `XUI_HOST` теперь требует явного указания протокола (`http://` или `https://`)
- **SSL verification:** Добавлен параметр `XUI_USE_SSL_CERT` для отключения проверки сертификатов

### 🔧 Технические детали

- Используется monkey-patching `requests.Session.request` для передачи `verify=False`
- Отключены warnings от urllib3 для самоподписанных сертификатов
- Патч применяется только при `XUI_USE_SSL_CERT=false`
- py3xui.Api не поддерживает параметр `use_ssl_certificate`, поэтому используется обходное решение

### 📝 Документация

- Улучшены комментарии в `.env.example` для HTTPS конфигурации
- Обновлена секция "Конфигурация" в README.md с примерами для HTTPS
- Добавлены примеры для разных сценариев (HTTP, HTTPS, кастомные пути)

### ⚠️ Breaking Changes

- **`XUI_HOST` теперь ДОЛЖЕН включать протокол**
  - ❌ Старый формат: `localhost:8443` или `127.0.0.1:2053`
  - ✅ Новый формат: `http://127.0.0.1:2053` или `https://localhost:8443/path/`
- Для HTTPS с самоподписанными сертификатами установите `XUI_USE_SSL_CERT=false`

### Примеры конфигурации

```bash
# HTTP (стандартный 3x-ui)
XUI_HOST=http://127.0.0.1:2053
XUI_USE_SSL_CERT=true

# HTTPS с самоподписанным сертификатом
XUI_HOST=https://localhost:8443/dostapn-3x-ui/
XUI_USE_SSL_CERT=false
```

## Формат версий

Проект использует [Semantic Versioning](https://semver.org/):
- MAJOR версия - несовместимые изменения API
- MINOR версия - новая функциональность (обратно совместимая)
- PATCH версия - исправления ошибок

## Типы изменений

- **Добавлено** - новая функциональность
- **Изменено** - изменения в существующей функциональности
- **Устарело** - функциональность, которая будет удалена
- **Удалено** - удаленная функциональность
- **Исправлено** - исправления ошибок
- **Безопасность** - исправления уязвимостей
